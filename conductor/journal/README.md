# Journal — Session Learning Capture

## Template

File name: `YYYY-MM-DD_<slug>.md`

```markdown
# Journal: <Title>

**Date**: YYYY-MM-DD
**Agent**: claude-main | gemini | codex | <agent-name>
**Tags**: tag1, tag2, tag3

## Context
What was the task? One sentence.

## Discovery
What did you learn that wasn't obvious before starting?

## What Worked
Approaches or patterns that succeeded.

## What Didn't Work
Approaches that failed and why.

## Future Guidance
What should the next session know about this area?
```

## Rules

1. Write a journal entry after any non-trivial session (debug, feature, audit, refactor)
2. Skip for purely conversational or informational sessions
3. Keep entries concise — focus on Discovery and Future Guidance
4. Tags are used for pattern detection (3+ same tag → propose standard)
5. Update INDEX.md with tags after each entry
