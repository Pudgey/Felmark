# AAS — Three-Layer Quality Review

Advocate (observe) → Adjudicate (test against standards) → Suitability (verify truth, hunt hallucination).

See `sops/AAS_SOP.md` for full protocol.

## Quick Reference

1. **Advocate**: Read code fresh, list what it actually does (not what it should do)
2. **Adjudicate**: Compare against project standards, flag deviations
3. **Suitability**: Verify correctness, hunt hallucinated logic, check edge cases

Use before merging, after completing missions, or when reviewing AI-generated code.
