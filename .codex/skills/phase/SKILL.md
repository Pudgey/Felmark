# Phase — Next Development Phase Proposal

Use `$phase` when the user wants a strategic next-step proposal for Felmark rather than immediate implementation.

Follow `conductor/skills/phase/SKILL.md`.

## What `$phase` does

1. Reads current product and conductor state
2. Proposes the next development phase
3. Explains why this phase should happen now
4. Defines entry criteria and exit criteria
5. Lists likely missions, risks, dependencies, and out-of-scope items
6. Ends with a clear approval question for the human

## Rules

- Planning only
- No implementation
- One recommended phase, not an unranked brainstorm dump
- Human approval is required before work begins

If no target area is specified, infer it from the current active product state and make that assumption explicit.
